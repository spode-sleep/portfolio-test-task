# Gleb Kosyrev — Portfolio

Full‑stack developer portfolio with contact form, AI‑powered confirmation messages, and an experimental 3D interactive view.

## Tech Stack

- **Frontend**: HTML, CSS (custom), vanilla JavaScript (TypeScript‑ready)
- **Backend**: TypeScript Serverless Function (Vercel)
- **AI**: OpenAI API (for personalized success messages)
- **Email**: Nodemailer (SMTP) — sends notifications to owner and confirmation copy to user
- **3D View**: p5.js (experimental interactive mode)
- **Deployment**: Vercel
- **Code generation**: Claude Code (assisted in initial code scaffolding)

## Features

- ✅ **Robust form validation** (client‑side and server‑side)
- ✅ **Page loader** — full‑screen loader on initial visit, waits for fonts and images before revealing the page
- ✅ **AI‑powered personalization** — OpenAI generates a unique, context‑aware success message after form submission
- ✅ **Dual email system** — detailed notification for the owner and a confirmation copy for the user (with consistent dark‑theme design)
- ✅ **Experimental 3D mode** — interactive first‑person view of the portfolio built with p5.js

## 3D Interactive View

The portfolio includes an experimental 3D mode (`portfolio-3d.html`) built with [p5.js](https://p5js.org/). It renders the portfolio content as interactive 3D cubes in a first‑person environment.

**Navigation:**

| Input | Action |
|-------|--------|
| `W A S D` | Move |
| Mouse | Look around |
| Click a cube | Show info |
| `ESC` | Release mouse |

> ⚠️ Highly experimental — best experienced on desktop with a mouse.

## AI Interaction

1. User fills out the contact form on the portfolio website.
2. Frontend validates the input (required fields, email format, etc.).
3. Data is sent to a Vercel serverless function.
4. The function:
    * Validates data on the server side.
    * Calls the **OpenAI API** to generate a personalized success message.
    * Sends two emails via Nodemailer:
        * A detailed notification to the site owner.
        * An automatic confirmation to the user.
5. The user sees the AI‑generated message on the website.

## Deploy to Vercel

### 1. Fork this repo

### 2. Import Project on Vercel

Go to [vercel.com/new](https://vercel.com/new) → Import from GitHub.

### 3. Set Environment Variables

In Vercel → Project → Settings → Environment Variables:

| Variable | Description |
|--------|-------------|
| `OPENAI_API_KEY` | Your OpenAI API key for generating success messages |
| `SMTP_HOST` | SMTP server address (e.g., `smtp.gmail.com`) |
| `SMTP_PORT` | SMTP port (`587` for TLS) |
| `SMTP_USER` | Your email address (e.g., your Gmail) |
| `SMTP_PASS` | App password for your email account |
| `OWNER_EMAIL` | Email address where form submissions are sent |
| `SKIP_MAIL` | Optional. Set to `true` to skip email sending (useful for testing) |

### Getting an OpenAI API Key

1. Go to [platform.openai.com/api-keys](https://platform.openai.com/api-keys).
2. Create a new API key.
3. Add it to your Vercel environment variables.

### Gmail App Password (for SMTP)

1. Enable 2‑Factor Authentication on your Google account.
2. Go to Google Account → Security → App Passwords.
3. Select "Mail" and your device, or choose "Other (custom name)".
4. Use the generated 16‑character password as your `SMTP_PASS`.

## AI Tools and Manual Adjustments

**Used AI tools:**

- **OpenAI API**: Generates the personalized "thank you" message shown to the user after successful form submission. The AI analyzes the user's name and message to craft a relevant and warm response.
- **Claude Code**: Assisted in generating the initial codebase, including the structure of the serverless function and parts of the frontend logic.

**Manually adjusted:**

The majority of the **styling and layout** required manual refinement:
* **Email templates**: The initial HTML/CSS for the emails was generated by AI, but the colors, spacing, and structure were manually adjusted for better readability and consistency across different email clients.
* **Frontend UI**: The positioning, fonts and colors of the form elements were fine‑tuned by hand to ensure an expressive and unique look.
* **3D view**: The p5.js interactive mode was iterated on manually.
